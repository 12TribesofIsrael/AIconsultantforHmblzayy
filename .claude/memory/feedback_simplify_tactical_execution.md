When Thomas asks how to *do* something hands-on on a platform (post a Reel, set up an automation, run a campaign step), the first response should be a single 2-minute action with named buttons — not the full multi-block blueprint, even if a polished blueprint exists in the repo.

**Why:** Apr 25 session — first response to "how do i start the remix funnel" pointed him at the full 5-block 2.5-hour blueprint in `docs/remix-overview.md`. He responded "that looks super complicated i dont know how to non of that" and we had to reset. He's strong on strategy/consulting but not deep on social-platform UI; comprehensive plans read as overwhelming when he just wants to ship one thing tonight. The fix that worked: stripped to a single IG Story sticker action with literal taps, then layered up only when he asked "what else."

**How to apply:**
- For "how do I do X" on a platform: lead with the single simplest path (1–2 min, named buttons, copy-paste-ready URLs/captions). Use the platform's native UI words ("paper-airplane icon," "Sticker → Link," "blue Next →").
- Save the full blueprint for "what's the whole funnel?" questions or after he's shipped the simple version once.
- If a multi-step plan is genuinely required, give the FIRST step only and offer to deliver the rest after he completes it.
- For tactical execution docs (the one-pager he prints/follows), prefer one stable filename (`docs/remix-playbook-today.md`) with a "today's facts" block at the top over per-day generated files he has to find each day.
- This is a layer beyond `feedback_present_only_optimal` — that rule says "don't surface A/B options"; this rule says "even when you've picked the optimal, scope it tiny first."
